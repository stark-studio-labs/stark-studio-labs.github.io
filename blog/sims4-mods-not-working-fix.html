Sims 4 Mods Not Working?
15 Fixes That Actually Work

Whether your mods vanished after an EA update, nothing shows up in your mods list, or the game crashes on load — there's almost always a specific fixable cause. Here's how to find it.

🔧 Covers PC, Mac & all game versions

There's nothing more frustrating than launching your game after an update and finding half your mods are gone or broken. It happens to every Sims 4 player who uses mods. The good news: it's almost always fixable, and it's almost always one of the same handful of causes.

Work through this list in order — start with the quick checks before jumping to the more involved fixes.

After every Sims 4 update

EA patches frequently break script mods (not just yours — everyone's). The first thing to check after an update is whether the mod creator has released a new version. Check their Patreon, Discord, or the original download page.

Quick Checks (Do These First)

  1. 1

    Make sure mods are actually enabled in settings

    Go to Main Menu → (⋯) Game Options → Other. Both "Enable Custom Content and Mods" and "Script Mods Allowed" must be checked. EA sometimes resets these after a major update.

  2. 2

    Fully restart the game

    Not just alt-tab or sleep mode — close The Sims 4 completely and reopen it. Mods won't load from a game that's already running.

  3. 3

    Check that files aren't still inside a ZIP

    Open your Mods folder and make sure you're not looking at a .zip file. The game cannot read compressed archives. You need to extract them first so you see actual .package or .ts4script files.

  4. 4

    Check your folder depth

    The Sims 4 only reads mods that are at most 5 subfolders deep inside the Mods folder. If you've organized things heavily, count your levels: Mods/Gameplay/Relationships/Author/Pack/file.package — that's 5 levels, right at the limit.

  5. 5

    Verify the mod is actually in the right folder

    The path should be: Documents/Electronic Arts/The Sims 4/Mods/. Not the game install folder, not Desktop, not Downloads. The Mods subfolder inside the Sims 4 save folder.

Script Mod Problems

  1. 6

    Script mods are a separate toggle from regular mods

    .ts4script files require the "Script Mods Allowed" checkbox specifically. The regular "Enable Custom Content and Mods" toggle alone won't run them. Check both boxes in Game Options.

  2. 7

    Script mods break more often than package mods

    When EA patches Python in the game (which happens with major updates), script mods that relied on the old Python behavior break until the author updates them. If a .ts4script mod stopped working right after an update — wait for the update from the creator. There's no workaround.

Finding the Broken Mod

  1. 8

    Use the 50% method to find a conflict

    If the game is crashing or behaving strangely, move half your mods to a temporary folder on your Desktop. Launch the game. If the problem is gone, the culprit is in the half you removed. Put back half of those, test again. Repeat until you find the exact mod. Takes 5–6 rounds for most collections.

  2. 9

    Read the lastException.txt file

    When a mod causes an error, the game creates a file at Documents/Electronic Arts/The Sims 4/lastException.txt. Open it in any text editor and look for the mod name — it's usually in the error traceback. That's your broken mod.

  3. 10

    Install Better Exceptions

    The Better Exceptions mod by TwistedMexi shows you a popup in-game that clearly names the mod causing the error instead of requiring you to dig through log files. Install it, replicate the problem, and it'll tell you exactly what's broken.

  4. 11

    Check for mod conflicts with MCCC

    If you have MC Command Center installed, it logs conflicts between mods. Open the MCCC settings menu and look for a conflicts section. It will list any mods that are overwriting the same game resources as each other.

Game Performance Issues

  1. 12

    Too many mods slow down loading

    Each .package file is loaded one by one when the game starts. 300+ mods will cause very long loading times. This isn't a bug — it's just how the game works. Consider trimming mods you don't actively use.

  2. 13

    Clear your cache files

    The Sims 4 builds cache files for mods. These sometimes become corrupted. Delete these files (the game will recreate them): localthumbcache.package, the cache folder, and cachestr folder — all inside Documents/Electronic Arts/The Sims 4/. Do NOT delete the Mods folder.

Mac-Specific Issues

  1. 14

    Mac security warnings blocking script mods

    macOS sometimes blocks .ts4script files from running because they contain Python code. If a script mod isn't loading on Mac, right-click the .ts4script file, choose Open, and click Allow in the security dialog.

  2. 15

    Verify your Mods folder path on Mac

    Mac Sims 4 saves go to ~/Documents/Electronic Arts/The Sims 4/Mods/. Note the tilde (~) — this means your home folder. If you have an unusual setup (multiple users, iCloud Documents sync), make sure the folder actually exists at this path.

Nuclear Option: Start Fresh

If you've tried everything and still can't figure out what's wrong, this always works:

  1. Move your entire Mods folder to your Desktop (don't delete it — just move it)
  2. Launch the game. It should work with no mods.
  3. Move mods back in small batches, testing after each batch
  4. When the problem comes back, the last batch you added contains the culprit

It's tedious, but it works every single time.

Prevention tip

Keep a backup copy of a clean, working Mods folder. When something breaks after an update, compare your current folder to the backup to identify what changed.
