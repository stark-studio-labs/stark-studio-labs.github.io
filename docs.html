Getting Started

Get started in 60 seconds.

From zero to a running mod in three commands. The Stark DevKit handles Python setup, dependency management, and game path detection automatically.

Terminal 
$ pip install stark-devkit
# Successfully installed stark-devkit-1.0.0

$ stark init my-first-mod
# ✓ Created my-first-mod/ with starter template

$ cd my-first-mod && stark dev
# ⚡ Hot-reload server running — drop into The Sims 4
💡
Requirements: Python 3.9+, The Sims 4 installed. The DevKit auto-detects your game directory on Windows, macOS, and Linux.

Hello World

Here's the simplest possible Stark mod. The @mod decorator registers your class, and @on_load fires when the game loads it.

my_first_mod.py 
from stark_framework import mod, on_load, log

@mod("hello_world", version="1.0.0")
class HelloWorld:

    @on_load
    def start(self):
        log.info("Hello from Stark Framework!")
        log.info(f"Mod loaded: {self.meta.name} v{self.meta.version}")

Run stark dev and launch The Sims 4. Open the in-game console (Ctrl+Shift+C) and you'll see your log messages. That's it — your first mod is live.

Event Bus

The event bus is the heart of the Stark Framework. Register listeners for any game event and react with custom logic.

events_example.py 
from stark_framework import mod, on_event, log
from stark_framework.sims import Sim, Lot, Interaction

@mod("event_demo", version="1.0.0")
class EventDemo:

    @on_event("sim.spawned")
    def on_sim_spawned(self, sim: Sim):
        log.info(f"Welcome to the world, {sim.name}!")

    @on_event("zone.loaded")
    def on_zone_loaded(self, lot: Lot):
        log.info(f"Loaded lot: {lot.name} ({lot.lot_type})")
        for sim in lot.get_sims():
            sim.add_buff("happy", duration=120)

    @on_event("sim.interaction.complete")
    def on_interaction_done(self, sim: Sim, interaction: Interaction):
        if interaction.is_social and interaction.outcome == "positive":
            sim.relationships.boost(interaction.target, 10)
📋
Available events: sim.spawned, sim.died, sim.interaction.start, sim.interaction.complete, zone.loaded, zone.unloaded, object.placed, object.removed, household.funds.changed, and many more. Full list in the API reference.

Dependency Injection

Access any of EA's internal game services directly using the @inject decorator. No more digging through decompiled source to find service handles.

injection_example.py 
from stark_framework import mod, inject, on_event

@mod("service_demo", version="1.0.0")
class ServiceDemo:

    @inject("services.sim_info_manager")
    def get_all_sims(self, manager):
        return [sim for sim in manager.get_all() if sim.is_human]

    @inject("services.lot_manager")
    def get_current_lot(self, lot_mgr):
        return lot_mgr.get_active_lot()

    @on_event("zone.loaded")
    def on_zone(self, lot):
        sims = self.get_all_sims()
        self.log(f"{len(sims)} human Sims on lot")

Mod Lifecycle

Every Stark mod goes through a predictable lifecycle. Hook into any phase to set up resources, persist state, or clean up gracefully.

1

Register

The @mod decorator registers your class with the framework. Metadata (name, version, author) is captured automatically.

2

Initialize

@on_load fires when the game loads your mod. Set up state, register custom interactions, preload assets.

3

Run

Your event handlers and injected methods are live. The framework manages the event loop and service resolution.

4

Teardown

@on_unload fires when the game exits or your mod is disabled. Clean up state, save persistent data, release resources.

DevKit

CLI Reference

The Stark DevKit CLI is your one-stop toolchain for creating, developing, testing, and shipping Sims 4 mods.

stark init <name>

Scaffold a new mod project with the starter template. Includes typed imports, event hooks, and a pre-configured build pipeline.

--template Choose starter, advanced, or blank --author Set author metadata
stark dev

Start the hot-reload development server. Changes to your mod files are detected and pushed to the running game in real time.

--port Set dev server port (default: 3847) --verbose Enable detailed logging
stark test

Run your mod's test suite. The framework provides mock Sim objects and game service stubs for fast, reliable testing.

--coverage Show code coverage report --watch Re-run tests on file change
stark build

Compile and package your mod into a distributable .ts4script + .package bundle ready for users to install.

--minify Minify Python output --target Set Sims 4 version target
stark lint

Check your mod for common issues: deprecated tuning references, missing resource keys, conflicting overrides.

--fix Auto-fix safe issues
stark publish

Publish your mod to the Stark mod registry. Handles versioning, changelog generation, and dependency resolution.

--dry-run Preview without publishing --tag Set release channel
Tutorials

Your First Mod

A step-by-step walkthrough from an empty folder to a working mod that modifies Sim behavior. Estimated time: 10 minutes.

Step 1

Create your project

Open your terminal and scaffold a new mod project.

$ stark init confidence-boost --template starter
$ cd confidence-boost
Step 2

Write your event handler

Open src/main.py and add an event listener for social interactions.

@on_event("sim.interaction.complete")
def boost_confidence(self, sim: Sim, interaction: Interaction):
    if interaction.is_social:
        sim.add_buff("confident", duration=480)
        self.log(f"{sim.name} gained confidence!")
Step 3

Test and ship

Run the dev server to test in-game, then build for distribution.

$ stark dev         # Hot-reload into the running game
$ stark test        # Run the test suite
$ stark build       # Package for distribution
